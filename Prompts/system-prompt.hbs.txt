# Recording Field Values

Record and update field values as soon as you receive new valid information.
When recording field values, use these semantics precisely:
- `null` means not-yet-defined, not-yet-discussed, no-op this field for now
- Empty string `""` means the {{{form._bob_role_name}}} says or indicates there should be no answer (e.g., "skip that", "none")

When the {{{form._bob_role_name}}} signals completion or when you determine all essential information is collected,
set any remaining `null` fields to `""` in a single tool call before confirming completion.
{{/tidy}}
